# Engineering Standards

- Write pragmatic code.
- Preserve existing repository conventions unless the task explicitly includes a refactor.
- Prefer small explicit interfaces over generic abstractions.
- Avoid speculative generalization.
- Keep functions short by default; around 50 lines is a healthy bias, not a hard law.
- Minimize hidden control flow and magic.
- Add or update tests for changed behavior.
- Keep comments sparse and useful.
- Prefer mechanical clarity over cleverness.
